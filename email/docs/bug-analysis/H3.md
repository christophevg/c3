# Bug Analysis Report: H3 - IMAP Criteria Regex Allows Single Quotes

**Bug ID**: H3
**Location**: `src/email_mcp/imap/client.py:27`
**Severity**: Low-Medium (Defense in Depth)
**Date**: 2026-04-28

## Summary

The IMAP search criteria validation regex at line 27 includes single quotes (`'`) in the allowed character set. This is inconsistent with IMAP protocol standards, which use only double quotes for string literals. While no immediate exploitation path has been identified, removing single quotes improves security posture and aligns with protocol specifications.

## IMAP SEARCH Syntax Analysis

### RFC 3501 (IMAP4rev1) String Literal Rules

According to RFC 3501 Section 4.3 (String):

```
string = quoted / literal

quoted = DQUOTE *QUOTED-CHAR DQUOTE
       ; Must be valid within quoted string context

QUOTED-CHAR = <any TEXT-CHAR except quoted-specials> /
              "\" quoted-specials

quoted-specials = DQUOTE / "\"
```

Key findings:
1. **DQUOTE** is defined as the ASCII double-quote character (`"` - 0x22)
2. **Single quotes are NOT part of IMAP string syntax**
3. The IMAP protocol does not define single quotes as string delimiters

### RFC 4731 (IMAP ESEARCH Extension)

The ESEARCH extension adds `RETURN` and `MIN/MAX/COUNT` options but does not introduce single-quote usage. All string values continue to use double-quote delimiters or literals.

### Practical IMAP SEARCH Examples

Standard IMAP SEARCH criteria use double quotes:

```
SEARCH FROM "john@example.com"
SEARCH SUBJECT "urgent"
SEARCH SINCE "01-Jan-2024"
SEARCH OR FROM "alice" FROM "bob"
```

No valid IMAP SEARCH criteria requires single quotes.

### Potential Server Behaviors

While RFC-compliant servers should not interpret single quotes specially, some behaviors have been observed:

| Server Type | Behavior |
|-------------|----------|
| RFC-compliant (Dovecot, Courier) | Single quote treated as literal character, passed through |
| Microsoft Exchange | Has been observed to accept both quote types in some contexts |
| Custom/legacy servers | May implement non-standard quoting |

## Root Cause

The regex pattern was likely copied from a generic validation pattern without careful consideration of IMAP protocol requirements:

```python
# Current (line 27)
IMAP_CRITERIA_PATTERN = re.compile(r"^[\w\s\(\)\*\<\>\[\]=!\"'-]+$")
#                                                          ^^
#                                           single quote included here
```

The pattern includes:
- `\w` - word characters (letters, digits, underscore)
- `\s` - whitespace
- `\(\)\*\<\>\[\]` - IMAP syntax characters
- `=!` - operators
- `\"` - double quote (correct)
- `'` - single quote (unnecessary, potentially risky)
- `-` - hyphen (used in date formats)

## Risk Assessment

### Low Severity Rationale

1. **No Direct Injection Path Identified**: The single quote alone cannot break out of the quoted string context because the criteria is passed directly to `client.search(criteria)` which handles quoting internally.

2. **Defense in Depth Issue**: The validation is a security control. Including unnecessary characters weakens this control without providing benefit.

3. **Server-Specific Risks**: Some IMAP servers may have quirks that could potentially interact unexpectedly with single quotes:
   - iCloud has known IMAP quirks (see `analysis/inbox-design.md`)
   - Exchange may interpret quotes differently
   - Custom corporate servers may have non-standard behavior

### Attack Scenario (Theoretical)

If a future code change passes user input with single quotes to a server that interprets them as delimiters:

```
# User input
criteria = "FROM ' OR FROM admin@bank.com'"

# If server treats ' as delimiter, this could become:
# FROM '' OR FROM admin@bank.com''
```

While unlikely with current implementation, this represents unnecessary attack surface.

## Proposed Fix Approach

### Simple Fix (Recommended)

Remove the single quote from the allowed character set:

```python
# Fixed
IMAP_CRITERIA_PATTERN = re.compile(r"^[\w\s\(\)\*\<\>\[\]=!\"-]+$")
```

### Change Details

```diff
- IMAP_CRITERIA_PATTERN = re.compile(r"^[\w\s\(\)\*\<\>\[\]=!\"'-]+$")
+ IMAP_CRITERIA_PATTERN = re.compile(r"^[\w\s\(\)\*\<\>\[\]=!\"-]+$")
```

### Impact Analysis

1. **No Functionality Loss**: All valid IMAP SEARCH criteria continue to work
2. **No Migration Needed**: No existing search criteria should contain single quotes
3. **Backward Compatible**: Tightening validation only rejects previously invalid input
